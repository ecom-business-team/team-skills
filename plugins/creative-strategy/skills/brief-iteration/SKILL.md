# /brief-iteration

Adapt a winning script into a new format/trend. Outputs a full brief with 5-10 hook options, hold copy, and CTA — ready for handoff.

## Usage

```
/brief-iteration
/brief-iteration [client name]
```

## When to Use

- You have a top-spending or top-performing script and want to reformat it
- A new creative trend is working and you want to port existing winning copy into it
- You're iterating on a batch and need hook variations fast

---

## Process

### Step 1: Gather Inputs

**Ask the user for:**

1. **The winning script** — paste the full copy
2. **New format/trend** — what's the execution style? (e.g., AI characterization, talking ingredient, UGC, POV, etc.)
3. **Reference batches** — any past batches that show the target format? (e.g., "match BATCH#47/63/64")
4. **Character POV** — if the format uses a character, who speaks? (e.g., Dutasteride, hair follicle, DHT, the product itself)
5. **Intensity level** — 1 (clean) to 3 (swears allowed, emotionally intense)
6. **Censor toggle** — should swears be written as f*ck or fully spelled out?

If the user already provided some of these in their message, confirm what you have and only ask for what's missing.

---

### Step 2: Map the Hold

**Do:**

Extract the "hold" — the shared body copy that runs after all hooks. This is typically:
- The product proof points (stats, ingredients, mechanism)
- The product name + format (e.g., "one daily chew")
- Social proof (doctors, users, reviews)
- Risk reversal (money-back guarantee)

Adapt the hold to fit the new format's voice and character if needed. If the format uses a character voice, the hold should either stay in character or transition to a narrator/product close naturally.

Keep all key proof points from the original. Do not remove claims, stats, or offer details.

**Output:** Hold copy block, labeled clearly, with note on voice (character or narrator)

---

### Step 3: Generate Hooks

**Do:**

Write 5-10 hook options in the new format. Each hook must:

- Open with immediate punch — funny, intense, or provocative from line 1
- Establish the character or POV within the first 1-2 lines
- End in a way that flows naturally into the hold opener
- Match the intensity level the user specified
- Be congruent with the hold — if the hold starts with "And when minoxidil joins the party," the hook must set up Dutasteride first

**Proven hook patterns to draw from (in order of strength):**
- Character calls out the viewer's inaction ("I've been here the whole time. What are we doing?")
- Character vs. problem ("I block 98% of what's killing you. You're welcome.")
- Self-aware frustration ("I'm your hair. I'm not okay.")
- Dry confidence ("That's not a flex. That's a fact.")
- Guilt + humor ("You do more for your gut than for me. I'm the one visibly struggling.")

Write hooks short. 3-5 lines max. No filler.

**Output:** Numbered hook list, each labeled with the POV/character

---

### Step 4: User Selects Direction

**Ask:**

> "Which hooks are closest? Pick 1-3 and I'll refine them, or tell me what direction to push."

Take their feedback and tighten the selected hooks. If they want a new direction, generate another round.

---

### Step 5: Assemble the Brief

**Do:**

Format the final output as a complete brief with these sections:

```
## [CLIENT] — Creative Brief
Batch: [batch number or "new"]
Format: [trend/format name]
Reference: [batch references]

---

### Concept
[1-2 sentences on what makes this iteration work strategically]

---

### Hook Options
[3-5 final hooks, each with:]
  - Character/POV label
  - Visuals note (in blue italic style: describe what's on screen)
  - Copy block

---

### Hold
[Shared body copy, labeled with voice: character or narrator]

---

### CTA
[Closing copy + offer + URL]

---

### Format Specs
| Element | Detail |
[Table with: format, style, characters, tone, CTA, offer]

---

### Key Proof Points
[Bulleted list of all claims, stats, and offer details from original script — nothing removed]
```

**Output:** Complete brief, ready for handoff or ClickUp entry

---

## Quality Check

Before delivering, verify:
- [ ] Every hook flows naturally into the hold opener — no awkward transitions
- [ ] All proof points from the original script are present in the hold or CTA
- [ ] Hook intensity matches the level the user specified
- [ ] Swear censor style is consistent (f*ck or fuck — not mixed)
- [ ] Character POV is consistent within each hook (don't switch mid-hook)
- [ ] Visuals notes are present for each hook option
- [ ] No filler lines — every line earns its place

If any check fails, fix before delivering.

---

## Notes

- **Hook-to-hold congruence is the #1 thing to get right.** The hold opener is fixed. Every hook must set it up.
- **Short hooks outperform long ones.** 3-5 lines. If it needs more, it's not punchy enough.
- **The character voice is the format.** AI characterization works because the ingredient/body part speaks with attitude. Don't dilute it with passive phrasing.
- **Keep all original proof points.** The winning script converted for a reason. Don't cut claims when reformatting.
- **Clients:** DFC Harness, DFC Beds (pet brand) / Keeps Hair Loss, Keeps TRT, Keeps ED (telehealth)
- **Proven character POVs:** hair follicle, DHT, Dutasteride, Minoxidil — use what fits the product
